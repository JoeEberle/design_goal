Design Goals

These design goals are applicable to all business solutions.

goal description
EXTENSIBILITY Extensibility is an ability to add or extend additional elements and features to the existing solution.
ACCESSIBILITY The ability for authorized users to easily access and use the solution from anywhere.
MAINTAINABILITY The ability to easily manage and update the solution.
SECURITY To secure the solution and data is only available to those who have appropriate access.
RELIABILITY The solution works consistently and can be counted on from a business perspective and performs as expected.
PERFORMANT The solution performs quickly enough to meet the business demand.
INTELLIGENT The solution learns or provides new insights.
CONSISTENCY Consistency means having solution work reliably and in the same manner every time.
SELF-DOCUMENTING The solution should be able to produce its own documentation.
SELF-TESTING Each component of the solution should be able to perform its own unit test.
TALKING The solution should be smart enough to explain itself to a non-technical human.
SIMPLE The solution should be lean and choose the simplest method of achievement.
PYTHONIC If the solution is built in python it is built using the python way.
NO CODE-LOW CODE The solution should contain the minimal code or no code required to achieve objective.
